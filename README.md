# Forensic Analysis & Intrusion Detection Toolkit

A small, runnable project demonstrating core techniques used in digital
forensics and network intrusion detection, built with **Python** (rapid
rule scripting and log analysis) and **C++** (performance-critical bulk
traffic processing) — mirroring how real-world tools like Volatility,
Snort/Suricata, and Zeek divide work between scripting and compiled code.

## What it does

| Component | Language | Purpose |
|---|---|---|
| `python/ids_engine.py` | Python | Rule-based network IDS: detects port scans, packet floods/DoS, and traffic to watchlisted ports. Rules are declarative, in `rules.json` — just like Snort/Suricata signature files. |
| `python/forensic_analyzer.py` | Python | Digital forensics on SSH auth logs: reconstructs a login timeline, detects brute-force attacks, flags whether an attack likely **succeeded**, and computes SHA-256 file-integrity hashes (tamper detection). |
| `cpp/fast_stats.cpp` | C++ | High-throughput per-source-IP traffic statistics (packet/byte counts, distinct destination ports, Shannon entropy of the port distribution) for datasets too large to comfortably crunch in pure Python. |
| `python/main.py` | Python | Orchestrates all three tools and produces one unified incident report (JSON + human-readable text). |
| `python/generate_sample_data.py` | Python | Generates synthetic network traffic and auth-log data with embedded attack patterns (port scan, flood, brute force) so the tools have something realistic to detect. |

## Quick start

```bash
# 1. Build the C++ component
cd cpp
make
cd ..

# 2. (Optional) regenerate sample data — already included in sample_data/
cd python
python3 generate_sample_data.py

# 3. Run the full pipeline
python3 main.py
```

Outputs land in `reports/`:
- `unified_report.json` — machine-readable, everything combined
- `unified_report.txt` — human-readable incident report
- `ids_alerts.json` — IDS-only output (also produced by running `ids_engine.py` directly)
- `forensic_report.json` — forensic-only output (also produced by running `forensic_analyzer.py` directly)

## Running components individually

```bash
cd python

# Network IDS only
python3 ids_engine.py ../sample_data/traffic.csv

# Forensic log analysis only (with file-integrity hashing)
python3 forensic_analyzer.py ../sample_data/auth.log --hash-dir ../sample_data

# Raw C++ traffic stats
../cpp/fast_stats ../sample_data/traffic.csv 15
```

## Detection logic, briefly

- **Port scan** (`SCAN-001`): a source IP contacts ≥20 distinct destination
  ports on the same host within a 60-second window.
- **Flood / DoS** (`FLOOD-001`): a source IP sends ≥100 packets to the same
  host:port within a 10-second window.
- **Watchlisted ports** (`PORT-001`): any traffic to ports commonly
  associated with malware/backdoors (4444, 31337, 12345, 6667, 1337).
- **Brute force** (forensic analyzer): ≥8 failed SSH logins from one IP
  within a 120-second window; if a successful login from that IP follows
  within 30 seconds, the finding is flagged as a likely **compromise**, not
  just an attempted attack.
- **Port-distribution entropy** (C++): scanning traffic spreads packets
  thinly across many ports, which drives Shannon entropy of the
  destination-port distribution way up (~8+ bits in the sample data) versus
  normal client traffic (~2.4 bits, concentrated on a handful of common
  ports).

## Extending this project

- Swap `rules.json` for your own rule definitions, or add new rule `type`s
  in `ids_engine.py` (e.g., DNS tunneling detection, geo-anomaly detection).
- Point `forensic_analyzer.py` at a real `/var/log/auth.log` (adjust the
  regex/year assumption in `parse_line` as needed).
- Feed `fast_stats.cpp` a real NetFlow/pcap-derived CSV for production-scale
  throughput testing — it's a single compiled binary with no dependencies.
- Add a `--baseline` / `--compare` mode to `hash_directory()` to diff two
  integrity manifests taken at different times (classic tamper detection).

## Notes

All data in `sample_data/` is **synthetic**, generated by
`generate_sample_data.py` for demonstration purposes only — no real network
captures or credentials are involved.
